Documentación de la API de Pokémon

Esta API permite obtener información sobre los tipos de Pokémon, un Pokémon aleatorio de un tipo específico, y el Pokémon con el nombre más largo de un tipo.

Endpoints Disponibles

  • http://127.0.0.1:5000/ - Inicio. Direcciona a la documentación de la API.
  • http://127.0.0.1:5000/documentation - Página de documentación de la API.
  • http://127.0.0.1:5000/pokemon/<nombre_pokemon>/type - Obtiene el tipo de un Pokémon dado su nombre (requiere autenticación).
  • http://127.0.0.1:5000/pokemon/random/<tipo_pokemon> - Devuelve un Pokémon aleatorio de un tipo específico (requiere autenticación).
  • http://127.0.0.1:5000/pokemon/longest-name/<tipo_pokemon> - Obtiene el Pokémon con el nombre más largo de un tipo específico (requiere autenticación).
  • http://127.0.0.1:5000/auth/login - Permite iniciar sesión y obtener un token JWT para autenticación.
  • http://127.0.0.1:5000/auth/register - Permite registrar un nuevo usuario en la API.

Entendimiento de los Endpoints

1. Obtener el tipo de un Pokémon (Requiere autenticacíón)

URL: http://127.0.0.1:5000/pokemon/<nombre_pokemon>/type

Método: GET

Descripción: Obtiene el tipo de un Pokémon dado su nombre.

Ejemplo de uso:
GET http://127.0.0.1:5000/pokemon/pikachu/type
Respuesta: {"Nombre del pokemon": "pikachu", "Tipo de Pokémon": ["electric"]}

2. Obtener un Pokémon aleatorio de un tipo específico (Requiere autenticacíón)

URL: http://127.0.0.1:5000/pokemon/random/<tipo_pokemon>

Método: GET

Descripción: Retorna un Pokémon aleatorio del tipo especificado.

Ejemplo de uso:
GET http://127.0.0.1:5000/pokemon/random/fire
Respuesta: {"Pokémon aleatorio": "charizard-mega-y", "Tipo de Pokémon": "fire"}

3. Obtener el Pokémon con el nombre más largo de un tipo específico (Requiere autenticacíón)

URL: http://127.0.0.1:5000/pokemon/longest-name/<tipo_pokemon>

Método: GET

Descripción: Devuelve el nombre del Pokémon más largo dentro del tipo especificado.

Ejemplo de uso:
GET http://127.0.0.1:5000/pokemon/longest-name/water
Respuesta: {"Nombre más largo": "blastoise-mega", "Tipo de Pokémon": "water"}

4. Registro de un nuevo usuario (Requiere método POST)

URL: http://127.0.0.1:5000/auth/register

Método: POST

Descripción: Permite registrar un nuevo usuario en la API.

        Ejemplo de uso:
        POST http://127.0.0.1:5000/auth/register
        Body:
        {
            "username": "nuevo_usuario",
            "password": "password123"
        }
        Respuesta: {"message": "Usuario registrado exitosamente"}

5. Inicio de sesión y obtención de token JWT (Requiere método POST)

URL: http://127.0.0.1:5000/auth/login

Método: POST

Descripción: Permite iniciar sesión y obtener un token JWT para autenticación.

        Ejemplo de uso:
        POST http://127.0.0.1:5000/auth/login
        Body:
        {
            "username": "usuario_existente",
            "password": "password123"
        }
        Respuesta: {"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}
